Registration with File Upload

This example shows how to register a new user with profile photo upload.

Must contain at least one lowercase letter, one uppercase letter, and one number
Supported formats: JPG, JPEG, PNG, GIF, BMP, WEBP. Max size: 5MB

API Endpoint Details

URL: POST /api/Authentication/register

Content-Type: multipart/form-data

Form Fields:

  • fullName (required): User's full name (letters and spaces only)
  • userName (required): Username (letters, numbers, underscores only)
  • email (required): Email address
  • password (required): Password (min 6 chars, must contain lowercase, uppercase, and number)
  • confirmPassword (required): Password confirmation
  • address (required): User's address (10-200 characters)
  • phoneNumber (required): Phone number
  • city (optional): City
  • country (optional): Country
  • profilePhoto (optional): Profile photo file (image, max 5MB)
  • role (required): User role (Customer, Admin, TechCompany, DeliveryPerson)

Response Format:

{
    "success": true/false,
    "message": "Response message",
    "data": "Additional data"
}

Notes:

  • The profile photo will be uploaded to the "profiles" folder in the assets directory
  • Supported image formats: JPG, JPEG, PNG, GIF, BMP, WEBP
  • Maximum file size: 5MB
  • If photo upload fails, registration will still proceed without the photo
  • The photo URL will be stored in the user's ProfilePhotoUrl field
  • All validation rules are enforced on both client and server side
